The following information reflects what is new in the edX Platform as of May 14, 2014. See previous pages in this document for a history of changes.

**************************
edX Documentation
**************************

You can access the `edX Status`_ page to get an up-to-date status for all
services on edx.org and edX Edge. The page also includes the Twitter feed for
@edXstatus, which the edX Operations team uses to post updates.

You can access the public `edX roadmap`_ for
details about the currently planned product direction.

The following documentation is available:

* `Building and Running an edX Course`_ 

  You can also download the guide as a PDF from the edX Studio user interface.

  Recent changes include: 

  * An updated `Enrollment`_ section to reflect that usernames or email
    addresses can be used to batch enroll students.

  * An updated `Grade and Answer Data`_ section that reflects new features on
    the problem Staff Debug viewer for rescoring, resetting attempts, and
    deleting student state.

  * An updated the `Staffing`_ section that explains the labeling differences
    between Studio and the LMS with respect to course team roles.

  * An updated `Assign Discussion Administration Roles`_ section to include a
    note about course staff requiring explicit granting of discussion
    administration roles.

  * A new `VitalSource E-Reader Tool`_ section.
    
  * Updated the `Add Files to a Course`_ section to include warnings about file
    size.


* `edX Data Documentation`_
  

* `edX Platform Developer Documentation`_

   Recent changes include: 

   * A new section on `Analytics`_ for developers.
  
  
* `edX XBlock Documentation`_
